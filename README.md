# Подключение новых изображений для главной страницы

Чтобы элементы и фон гарантированно работали на локальном сервере, все новые картинки нужно класть в:

- `Site/assets/img/`

## Ожидаемые файлы в `Site/assets/img/`

### Фоны
- `heroBg.webp`
- `heroBgMobile.webp`

### Элементы HERO
- `ball.webp`
- `ball2.webp`
- `boots.webp`
- `arrow1.webp`
- `crown.webp`

### Элементы блоков
- `program.webp`
- `location.webp`
- `rewies.webp`

### Элементы SITEBG
- `conus.webp`
- `money.webp`
- `coach.webp`
- `swistl.webp`
- `TV.webp`
- `book.webp`
- `chess.webp`
- `Chart.webp`
- `test.webp`

## Примечания
- Бинарные файлы агентом не создавались и не изменялись.
- Если какого-то файла нет, браузер покажет пустой слот/битую иконку — проверьте имя и регистр букв.
- Рекомендованный формат для финальной оптимизации: `webp` (после согласования визуала).
